Mister Donkey

Product Building · API Integration · AI Summaries · UX · Deployment · Brand Voice

/assets/img/mister-donkey-visual.jpg

A personality-driven weather assistant combining product thinking, frontend and backend deployment, location handling, API integration, and AI-generated summaries.

What it is

Mister Donkey is a weather assistant with a deliberately opinionated voice. It turns location-aware weather data into a usable forecast experience, then layers in concise AI-generated summaries that make the product feel specific rather than generic.

The personality is part of the interface, but the product still has to do the ordinary work well: resolve location, call weather services, handle edge cases, render cleanly, and deploy reliably.

Why it belongs in the portfolio

This project demonstrates practical product-building across the whole surface area: concept, brand voice, data integration, frontend behavior, backend handling, deployment, and maintenance. It is small enough to be legible and complete, but real enough to require production decisions.

What it demonstrates

  • Turning a simple utility into a differentiated product experience.
  • Using API data and AI summaries without letting the AI become the whole product.
  • Designing a voice that is memorable while keeping the user task clear.
  • Shipping and maintaining a live web application.
